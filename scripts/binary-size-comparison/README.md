# Binary Size Comparison

This directory is the reproducible, GitHub-linkable evidence for the sub-2k
Koru hello-world binary.

Run:

```sh
scripts/binary-size-comparison/compare.sh
```

The Koru source is generated by
[`compare.sh`](compare.sh):

```koru
~import "$std/io"
~std.io:print.blk {
    Hello, World!
}
```

That syntax is not special-cased in the benchmark. It goes through Koru source
block parsing:

- [`hasSourceBlock`](../../src/parser.zig) detects a top-level `{` without a
  prior top-level `=`.
- [`parseImplicitSourceBlock`](../../src/parser.zig) captures the block as raw
  text, dedents it, and stops at the standalone closing brace.
- [`createImplicitSourceInvocation`](../../src/parser.zig) attaches that text as
  an `ast.Source` argument to the invocation.

The sub-2k path is the `print.blk|raw_posix` transform in
[`koru_std/io.kz`](../../koru_std/io.kz). On Linux and macOS,
`std.build:variants` routes `std.io:print.blk` to `raw_posix`, which emits
inline `std.posix.write` calls instead of `std.debug.print`.

So the actual path is:

```text
source block -> ast.Source -> print.blk transform -> raw_posix variant -> inline posix.write -> ReleaseSmall stripped ELF
```

The size build is:

```sh
zig build-exe -target x86_64-linux -O ReleaseSmall -fstrip \
  -fno-unwind-tables -z norelro -fno-builtin output_emitted.zig \
  -femit-bin=hello_koru
```

On the checked local toolchain this produces a Koru ELF around 1.7 KB, matching
the Zig raw `posix.write` baseline.
