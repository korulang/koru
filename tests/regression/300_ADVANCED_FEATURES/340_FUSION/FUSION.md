# Event Fusion - EXPERIMENTAL

## Overview

Event fusion is an experimental optimization pass that detects chains of transitively pure event calls and transforms them into optimized variants, preserving the original implementations for verification and debugging.

**Status**: Phase 2 In Progress (Transformation with Provenance Tracking)

---

## Architecture: Self-Governing Optimization Pass

Fusion follows Koru's **self-governing compiler pass pattern**:

### How It Works

1. **Lives in default compiler pipeline** - No user import needed (unlike CCP)
2. **Compile-time AST transformation** - Not runtime instrumentation
3. **Flag-controlled execution** - Checks `--fusion` flag to enable/disable
4. **Self-contained module** - `koru_std/optimizations/fusion.kz`

### Evolution Path

**Phase 2a (Experimental - Current):**
```bash
koruc --fusion input.kz  # Opt-in, explicit
```

**Phase 2b (Trusted - Future):**
```bash
koruc input.kz              # Fusion runs by default
koruc --no-fusion input.kz  # Opt-out if needed
```

**Phase 2c (Mature - Later):**
```bash
koruc input.kz  # Fusion is just part of compilation
```

### Comparison with CCP

| Aspect | CCP | Fusion |
|--------|-----|--------|
| **Type** | Runtime instrumentation | Compile-time optimization |
| **Import** | Required (`~import "$std/ccp"`) | Not needed (in pipeline) |
| **Activation** | Import + `--ccp` flag | `--fusion` flag only |
| **What it does** | Adds profiling hooks to generated code | Transforms AST before codegen |
| **When it runs** | Program execution | Compilation |

## Phase 1: Detection (Complete ✅)

### Fusion Detector (`src/fusion_detector.zig`)
- Analyzes AST for fusion opportunities
- Uses purity tracking to validate fusable chains
- Reports detected opportunities during compilation
- Foundation for Phase 2 transformation

**Detection Algorithm:**
1. Register all events in the AST
2. Walk each proc's inline flows
3. Build chains of event invocations
4. Check if ALL events in chain are transitively pure
5. Report chains of 2+ pure events as fusion opportunities

**Key Insight:** Only transitively pure event chains can be safely fused!

**Output:**
```
🔥 FUSION OPPORTUNITIES DETECTED:
   Found 2 fusable chain(s) with 4 total events
   📍 In calculate: add -> multiply (2 events)
```

## Phase 2: Transformation with Provenance (In Progress 🚧)

### The Variant Strategy

Instead of replacing code, fusion **generates optimized variants** that coexist with originals:

```koru
// Original code - preserved as |naive variant
~proc calculate|naive {
    const sum = ~add(x: 10, y: 20);
    | done s |> ~multiply(x: s.result, factor: 3)
    | done m |> return .{ .done = .{ .result = m.result } };
}

// Generated by fusion pass - added as |fused variant
~proc calculate|fused {
    // Fused: add(10, 20) -> multiply(result, 3)
    const result = (10 + 20) * 3;
    return .{ .done = .{ .result = result } };
}
```

**Benefits:**
- ✅ Original code preserved (debuggable, readable)
- ✅ Optimized code testable (property test equivalence)
- ✅ Both variants benchmarkable (measure actual speedup)
- ✅ Can toggle optimization at build time
- ✅ AST doesn't lose information

### Provenance Tracking

Each optimized variant tracks its origin:

**AST Structure:**
```zig
pub const ProcDecl = struct {
    // ... existing fields ...

    // EXPERIMENTAL: Optimization provenance
    derived_from: ?*const ProcDecl = null,      // Pointer to source proc
    optimization_applied: ?[]const u8 = null,   // "fusion", "inline", etc.
};
```

**Example:**
```
calculate|fused:
  derived_from: &calculate|naive
  optimization_applied: "fusion"
```

**Enables:**
- 🔍 **Interactive debugging** - Trace back to original code
- 📊 **Property testing** - Verify optimized ≈ original
- 📈 **Benchmarking** - Compare performance
- 🤖 **AI-driven optimization** - Future CCP integration

### Fusion Transformer (`koru_std/optimizations/fusion.kz`)

**Structure:**
```koru
~event optimize.fusion {
    ast: ProgramAST,
    enable_fusion: bool
}
| optimized { ast: ProgramAST, opportunities_count: u32 }
| unchanged {}

~proc optimize.fusion {
    // Self-governing: Check if enabled
    if (!enable_fusion) {
        return .{ .unchanged = .{} };
    }

    // Detect fusion opportunities
    const opportunities = ~detect_fusion(ast: ast);

    // Transform: Generate |fused variants
    for (opportunities) |opp| {
        const fused_proc = generate_fused_variant(opp);
        fused_proc.derived_from = opp.original_proc;
        fused_proc.optimization_applied = "fusion";
        ast.add_proc(fused_proc);
    }

    return .{ .optimized = .{
        .ast = ast,
        .opportunities_count = opportunities.len
    } };
}
```

### Pipeline Integration

**File:** `koru_std/compiler.kz`
```koru
~proc compiler.coordinate.default {
    // ... existing passes (parse, shape check, purity) ...

    // Fusion optimization (self-governing)
    ~optimize.fusion(ast: ast, enable_fusion: config.fusion)
    | optimized |> std.debug.print("🔥 Fusion: {} opportunities fused\n", .{optimized.opportunities_count})
    | unchanged |> {}  // Skip if disabled

    // ... continue with (possibly optimized) AST ...
}
```

```koru
const a = ~add(x: 10, y: 20);        // Pure
```

```koru
~test.property(name: "fusion equivalence: calculate") {
    const test_cases = generate_test_inputs();

    for (test_cases) |input| {
        const naive_result = ~calculate|naive(input);
        const fused_result = ~calculate|fused(input);

        assert(equal(naive_result, fused_result));
    }
}
```

## Philosophy

**Start SAFE, build TRUST, then OPTIMIZE AGGRESSIVELY:**

1. **Phase 1 (Detection)** - Prove we can identify opportunities
2. **Phase 2a (Opt-in)** - Generate variants, require `--fusion` flag
3. **Phase 2b (Trusted)** - Flip to default, allow `--no-fusion` opt-out
4. **Phase 2c (Standard)** - Just part of compilation

**Key Principles:**
- **Never delete original code** - Variants coexist
- **Always verify equivalence** - Property tests prove correctness
- **Track provenance** - Know where optimizations came from
- **Let it fail loudly** - Bad optimizations are visible
- **Measure everything** - Benchmark actual gains
